# PRD: ИИ-Агент «Менеджер клиентского сервиса» — EdPalm

**Версия:** 1.0
**Дата:** 2026-03-02
**Статус:** Draft

---

## 1. Контекст и цель

EdPalm — онлайн-школа семейного образования (1–11 классы). Команда клиентского сервиса вручную обрабатывает онбординг, типовые вопросы, напоминания по документам и оплатам. Это создаёт нагрузку и задержки в ответах.

**Цель V1:** Запустить ИИ-агента поддержки, который сопровождает клиента на всём пути обучения — отвечает на вопросы, напоминает о важных событиях и эскалирует сложные случаи на команду КС.

**Метрики успеха V1:**
| Метрика | Цель |
|---|---|
| Доля обращений закрытых без эскалации | ≥ 65% |
| Среднее время ответа | ≤ 5 сек |
| NPS после взаимодействия с агентом | ≥ 4 из 5 |
| Охват триггерных уведомлений | 100% клиентов с Telegram |
| Uptime | 99.5% |

---

## 2. Место агента в экосистеме EdPalm

Агент поддержки — это та же платформа «Эврика», что и агент-продажник, но с другой ролью. Система определяет роль по зданию на карте портала:

| Здание | Роль Эврики |
|---|---|
| Магазин | Менеджер по продажам |
| Служба поддержки | Менеджер клиентского сервиса |

Для пользователя это выглядит нативно — один и тот же чат-интерфейс, одно имя, разный контекст.

## 4. Архитектура

Агент поддержки использует ту же инфраструктуру что и агент-продажник:

```
┌──────────────────────────────────────────────┐
│              КЛИЕНТСКИЕ ОБОЛОЧКИ             │
│  Портал (здание КС) │ Telegram │ Ссылка      │
└─────────────────┬────────────────────────────┘
                  │
         ┌────────▼────────┐
         │  React Chat UI  │  (тот же SPA)
         └────────┬────────┘
                  │ REST / SSE
         ┌────────▼────────┐
         │  Python FastAPI │  (Agent Core)
         │  роль: Support  │
         └───┬────┬────┬───┘
             │    │    │
         OpenAI  Supabase  Integrations
         GPT-4o            - amoCRM (Service pipeline)
                           - DMS API
                           - Telegram Bot (уведомления)
```

**Стек** — идентичен агенту-продажнику:

| Слой | Технология |
|---|---|
| Frontend | React 19 + Vite + @twa-dev/sdk |
| Backend | Python 3.12 + FastAPI |
| БД | Supabase (PostgreSQL + pgvector) |
| LLM | OpenAI GPT-4o |
| RAG | pgvector + OpenAI text-embedding-3-small |
| CRM | amoCRM, воронка Сервиса (`10490518`) |
| Данные клиента | DMS API (Go-бэкенд EdPalm) |
| Уведомления | Telegram Bot API (`miniapp_edpalm_bot`) |
| Деплой | Backend: VPS. Frontend: Vercel |

## 5. Аутентификация

Идентична агенту-продажнику:
1. **Портал → Агент:** JWT от PHP-портала (TTL 15 мин) → бэкенд верифицирует → профиль из DMS
2. **Telegram Mini App:** `initData` HMAC-SHA256 → поиск по `telegram_id` в DMS
3. **Внешняя ссылка:** подписанный URL-токен (TTL 48ч)

## 6. Функциональные требования

### 6.1. Онбординг после покупки

**Триггер:** событие «оплата прошла» из DMS (вебхук)

**Флоу:**
1. Агент пишет клиенту в Telegram: приветствие, поздравление с началом обучения
2. Краткая инструкция: как войти на платформу, где расписание, как связаться с поддержкой
3. Через 24ч после доступа: «Всё получилось войти? Есть вопросы?»
4. Если клиент не отвечал 48ч — алерт менеджеру КС

---

### 6.2. FAQ и техническая помощь

**Триггер:** клиент пишет сам

**Примеры запросов:**
- «Не грузится видео»
- «Забыл пароль»
- «Где найти домашнее задание?»
- «Как связаться с преподавателем?»

**Флоу:**
1. RAG-поиск по базе знаний EdPalm
2. Агент даёт конкретный ответ с инструкцией
3. Если не решилось — эскалация на менеджера КС с описанием проблемы
4. После решения: «Вопрос решён? Оцените помощь от 1 до 5» (NPS)

---

### 6.3. Персонализация по тарифу и статусу

Перед каждым ответом агент проверяет через DMS:
- Тариф клиента → адаптирует ответ
- Статус зачисления → отвечает на вопросы о документах и приказе
- Статус обучения → успеваемость, пройденные модули

**Примеры поведения:**
| Ситуация | Действие агента |
|---|---|
| На «Базовом», спрашивает про индив. урок | «На вашем тарифе индивидуальные уроки не включены — можно докупить отдельно» |
| На «Премиум», то же | «Давайте назначим время — напишите удобный слот» |
| «Когда будет приказ о зачислении?» | Проверяет статус в DMS → отвечает по факту |

---

### 6.4. Триггерные уведомления — клиентам

Агент инициирует диалог через Telegram Bot по расписанию:

| Триггер | Когда | Сообщение |
|---|---|---|
| Напоминание об оплате | За 3 дня, за 1 день, в день X | Персональное: сумма, ссылка на оплату |
| Старт занятий | За 1 день | Расписание на завтра, ссылка на платформу |
| Дедлайн домашнего задания | За 1 день | Напоминание с названием задания |
| Документы не загружены | На 3-й день без загрузки | Напоминание + инструкция |
| После зачисления | День приказа | Поздравление + следующие шаги |

---

### 6.5. Триггерные уведомления — менеджерам КС

| Триггер | Алерт |
|---|---|
| Клиент игнорирует агента 48ч (документы / оплата) | Уведомление менеджеру: имя клиента, что ждём, сколько дней |
| Падение успеваемости ученика (данные из DMS) | «Ученик X не сдал 2+ аттестации — рекомендуем звонок заботы» |
| Посещаемость < 50% за 2 недели | Алерт менеджеру с деталями |

---

### 6.6. Модуль ГИА (ОГЭ/ЕГЭ)

Для учеников 9–11 классов.

**Функции:**
- Информация о сроках подачи заявлений, датах экзаменов, правилах апелляций
- Чек-листы документов для выпускников
- FAQ по формату: что брать с собой, как заполнять бланки, правила поведения
- Напоминания о ключевых дедлайнах ГИА по расписанию

**Зависимость:** требует отдельного блока базы знаний по ГИА — актуальная информация за учебный год.

---

### 6.7. Аналитика и тегирование

**Автотегирование диалогов:**
Каждому диалогу автоматически присваивается тег по теме:
`#оплата` `#документы` `#ошибка_платформы` `#успеваемость` `#гиа` `#расписание` `#другое`

**NPS после каждого обращения:**
«Вопрос решён? Оцените помощь: 1 — 2 — 3 — 4 — 5»

**Еженедельный AI-отчёт для руководителя КС:**
- Топ тем обращений за неделю
- Количество эскалаций и причины
- NPS-динамика
- Инсайты: «45 клиентов не смогли найти X — рекомендация: улучшить навигацию»

---

### 6.8. Эскалация на менеджера КС

1. Агент: «Соединяю вас с менеджером — он ответит в ближайшее время»
2. В amoCRM (воронка Сервиса `10490518`): создаётся тикет с историей диалога, тегом и саммари проблемы
3. Менеджер КС получает уведомление в Telegram: имя клиента, суть вопроса, ссылка на тикет
4. Поле ввода блокируется до ответа менеджера

---

### 6.9. Guardrails

| Ситуация | Действие |
|---|---|
| Вопрос не по базе знаний | «Уточню у специалиста» → эскалация |
| Явный негатив клиента | Немедленная эскалация |
| Вопрос о конкретном преподавателе | Только по данным из базы |
| Вопрос о персональных данных других | Отказ |
| Попытка prompt injection | Игнорирование, ответ в роли |

## 7. Инструменты агента (Function Calling)

| Инструмент | Описание |
|---|---|
| `search_knowledge_base(query)` | RAG-поиск по базе знаний EdPalm |
| `get_client_profile(user_id)` | Профиль из DMS: тариф, статус, имя |
| `get_student_status(student_id)` | Статус зачисления, успеваемость, посещаемость через DMS |
| `get_amocrm_ticket(contact_id)` | Текущий тикет в воронке Сервиса |
| `create_amocrm_ticket(data)` | Создать тикет при эскалации |
| `send_telegram_notification(user_id, text)` | Исходящее уведомление клиенту |
| `notify_manager_cs(alert_data)` | Алерт менеджеру КС в Telegram |
| `schedule_reminder(user_id, date, template)` | Запланировать уведомление |
| `collect_nps(conversation_id)` | Запустить NPS-опрос после закрытия |
| `tag_conversation(conv_id, tags)` | Присвоить теги диалогу |
| `generate_weekly_report()` | Сформировать отчёт для руководителя КС |
