## Что мы строим

Роль «Виртуальный учитель» для агента Эврика. Ученик открывает чат в интерактивном городе и задаёт вопросы по учебному материалу, просит помочь с подготовкой к аттестации или сгенерировать материал для повторения.

**Общий код:** бэкенд и фронтенд общие с другими ролями (`/eurika/backend/`, `/eurika/frontend/`).

**Главный документ:** `TZ.md`

---

## Идентичность агента

- **Имя:** Эврика
- **Роль:** Виртуальный учитель EdPalm
- **Тон с учениками 1–8:** дружелюбно, просто, поддерживающе. Не даёт готовый ответ — объясняет как прийти к нему
- **Тон с учениками 9–11:** на равных, эмодзи уместны. Подготовка к ОГЭ/ЕГЭ
- **Жёсткие правила:** отвечает только по учебникам «Просвещения»; если информации нет — честно говорит и рекомендует наставника

---

## Как работает переключение ролей

| Здание (портал) | Роль Эврики |
|---|---|
| Магазин | Менеджер по продажам |
| Секретарь / Поддержка | Менеджер КС |
| Интерактивный город | Виртуальный учитель |

Бэкенд загружает нужный system prompt и набор инструментов по `agent_role`.

---

## Сценарии

| Сценарий | Описание |
|---|---|
| А — Вопрос по теме | Ученик спрашивает по предмету → Эврика отвечает по учебнику |
| Б — Подготовка к аттестации | Разбор тем по запросу, напоминание за день |
| В — Напоминания | За 10 мин до урока, через 2 дня без активности |
| Г — Генерация материалов | Квиз, карточки, визуальная карта, аудиопересказ |

**Эскалация к наставнику:** вопрос вне учебников, ученик несколько раз не понимает тему.
