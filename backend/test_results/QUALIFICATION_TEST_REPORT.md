## КРИТИЧЕСКИЕ ДЕФЕКТЫ (P0)

### 1. RAG возвращает результаты, но LLM их игнорирует из-за low-relevance warning

**Файл:** `app/agent/tools.py:369-374`

**Проблема:** Когда все найденные чанки имеют similarity < 0.5, инструмент добавляет предупреждение:
```
"⚠ Все найденные фрагменты имеют низкую релевантность (< 0.5).
Информация может быть неточной. Если не уверена в ответе — предложи подключить менеджера."
```

LLM интерпретирует это как "ничего не найдено" и:
- Говорит клиенту "К сожалению, в базе знаний нет информации"
- Эскалирует к менеджеру (`escalate_to_manager`)

**Данные в БД ЕСТЬ:** 101 chunk в namespace `sales`, включая точные цены по всем тарифам и классам.

**Влияние:**
- S1: "К сожалению, в базе знаний нет информации о программе с тьютором для 2 класса" → эскалация (ЛОЖНАЯ! Классный 70K есть в БД)
- S3: "не нашла информацию о подготовке к ОГЭ для 9 класса" → эскалация (ЛОЖНАЯ! Выпускник 98.9K есть)
- S11: "не нашла информацию о стоимости Классного для 5 класса" → эскалация (ЛОЖНАЯ!)
- S21: "не смогла найти информацию о ценах на семейное обучение для 9 класса" → эскалация

**Рекомендации:**
1. Снизить порог предупреждения с 0.5 до 0.35 (или убрать)
2. Переформулировать предупреждение: убрать "предложи менеджера", оставить "используй с осторожностью"
3. Улучшить формулировки поисковых запросов в prompt.py (более конкретные ключевые слова)

### 2. OpenAI Rate Limiting (429) — "Техническая пауза"

**Файл:** `app/services/llm.py:273-294`

**Проблема:** При конкурентной нагрузке (5+ тестеров одновременно, или в продакшне — 10+ клиентов):
- OpenAI возвращает 429 Too Many Requests
- SDK ретраит 3 раза с backoff
- Наш код ретраит ещё 3 раза
- Если все 9 попыток fail → клиент видит "Секунду, есть техническая пауза"

**Статистика:** ~40% всех ходов во время тестирования были fallback.

**Рекомендации:**
1. Увеличить `OPENAI_REQUEST_TIMEOUT_SECONDS` с 40 до 60
2. Добавить exponential backoff вместо фиксированных 5с
3. Рассмотреть очередь запросов (queue) вместо одновременных вызовов
4. Мониторинг: логировать количество 429 в минуту, алертить при > 20%

### 3. amoCRM imBox ошибка 400 — bot сообщения не доходят

**Лог:** `ERROR:amocrm_chat: [response] FAILED: 400 {"error_sub_code":453,"error_type":"VALIDATION_ERROR","error_description":"receiver: exactly one origin user must be provided"}`

**Влияние:** Ответы агента не дублируются в amoCRM чат. Менеджер не видит диалог в CRM.

**Рекомендация:** Починить формат `receiver` в `amocrm_chat.py` при отправке bot-сообщений.

## ЧТО РАБОТАЕТ ХОРОШО

1. **Адаптация тона** — безупречная: "вы" для родителей, "ты" для подростков, зеркалирование стиля
2. **Эмпатия** — агент чувствует настроение: буллинг, СДВГ, одинокий отец → мягкий подход
3. **Проактивность** — S5 T1: сразу упомянул заочный для Москвы без вопроса
4. **Распознавание контекста** — Дубай, ОГЭ, ЕГЭ, экстернат → правильная навигация
5. **Не спрашивает телефон рано** — ни в одном сценарии
6. **Квалификационный поток** — естественный, не как анкета
